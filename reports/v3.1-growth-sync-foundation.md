# v3.1 自成长与多端同步基础报告

生成日期：2026-07-09

## 1. 本轮目标

- 在 v3.0 的安全、备份、学习领域和知识来源基础上，继续补齐“知识库自成长不污染正式库”的门禁。
- 将错题本从简单列表推进为能力画像，为后续复习推荐和自进化提供结构化依据。
- 建立 Android 与 Windows 共用的增量同步 API 骨架，先保证协议稳定，再做客户端实现。

## 2. 已完成能力

### 2.1 知识成长评估门禁

- 新增 `GrowthGateService`。
- 候选资料来源发布前检查：
  - 标题是否存在；
  - 许可是否明确；
  - SHA256 内容哈希是否有效；
  - URL 是否命中登录、付费、验证码等风险关键词。
- 候选知识节点发布前检查：
  - 节点名是否存在；
  - 难度、考试重要性、工程重要性是否在 1-5 星范围；
  - 是否包含公式符号并需要人工渲染复核。
- 发布接口：
  - `POST /api/v1/sources/{source_id}/publish`
  - `POST /api/v1/knowledge-nodes/{node_id}/publish`

### 2.2 错题能力画像

- 新增 `AbilityProfileService`。
- 按学科、知识点、难度和错误类型聚合错题。
- 错误类型初步分为：
  - `concept`：概念理解问题；
  - `formula`：公式、单位、符号问题；
  - `calculation`：数值计算问题；
  - `reading`：审题遗漏问题；
  - `operation`：软件或操作步骤问题；
  - `unknown`：暂不能判断的问题。
- 新增接口：
  - `GET /api/wrong-book/ability-profile`

### 2.3 多端同步 API 骨架

- 新增 `SyncService` 和同步事件表。
- Android 可推送离线事件：
  - `wrong_question`
  - `photo_task`
  - `study_progress`
  - `note`
- Windows/Android 可拉取：
  - 学习领域列表；
  - 能力画像；
  - 服务端收到的增量事件。
- 新增接口：
  - `POST /api/v1/sync/push`
  - `POST /api/v1/sync/pull`

## 3. 为什么这样写

- 成长门禁先于自动发布：AI 生成内容、联网候选资料和 OCR 文本都可能有错误，默认进入候选态更安全。
- 能力画像先用本地 JSONL：当前错题链路已经稳定记录字段，先聚合现有数据，比直接重构错题数据库风险低。
- 同步先做事件协议：Android 端后续离线拍照、刷题、记笔记都可统一转成事件，Windows 主库负责统一合并。
- 不在 Android 保存 DeepSeek Key：移动端只同步资料、错题和任务，云模型调用仍通过电脑主库代理。

## 5. 常见问题判断

- 发布资料返回 `needs_review`：通常是许可不明确或公式风险未人工复核，不应直接绕过。
- 同步事件进入 `rejected`：事件类型不在允许列表中，需要扩展 `ALLOWED_SYNC_TYPES` 并补测试。
- 能力画像中出现 `未知学科` 或 `未知知识点`：说明历史错题缺少 `subject` 或 `knowledge_point` 字段，需要从题目会话或前端提交链路继续补齐。
- Android 拉取事件过多：后续需要按 `server_received_at` 保存游标，客户端每次传 `since`。

## 6. 下一轮建议

1. 把错题能力画像接到网页端，替代当前只显示“错误最多”的粗统计。
2. 给同步 API 增加设备配对、局域网/USB 模式提示和冲突解决策略。
3. 为 Android Flutter 工程补齐原生目录、权限、离线 SQLite 和拍照队列。
4. 为 Windows 客户端选择 WebView2/Tauri 方案，负责启动后端、备份、恢复和更新。
5. 将候选知识内容评估扩展为公式渲染截图检查、来源一致性抽测和检索污染回归测试。
