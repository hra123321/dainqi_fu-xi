# v2.6 自定义学科与本地向量入库阶段报告

## 改了什么

- 增加后端学科注册表：用 SQLite 保存自定义学科、知识点、入库任务状态。
- 增加新增学科页面：用户输入学科名后，网页自动创建学科并启动入库。
- 增加联网资料发现服务：优先使用 `SEARXNG_URL`，未配置时使用 DuckDuckGo Instant Answer 做有限候选发现。
- 增加 DeepSeek 固定提示词：`subject_discovery.txt` 负责把搜索摘要整理为知识点 JSON。
- 增加 Chroma 学科元数据：入库片段带 `subject_id/source_id/topic/license/hash`，检索可按学科过滤。

## 为什么这样写

- 学科不能继续写死在 JS；否则新增学科刷新后会丢失，也无法被后端刷题链路识别。
- SQLite 是本地单文件数据库，适合保存小规模业务状态，不需要额外安装数据库服务。
- LLM 不直接“联网搜索”；搜索适配器负责找候选资料，DeepSeek 负责结构化整理，能降低幻觉风险。
- 新增学科不生成独立 HTML 文件，而是通过动态 API 自动出现在网页中，维护成本更低。

## 如何验证

- 编译检查：`.\venv\Scripts\python.exe -m compileall -q app main.py tests`
- 学科服务测试：`.\venv\Scripts\python.exe tests/test_subjects.py`
- 学科 API 测试：`.\venv\Scripts\python.exe tests/test_subject_api.py`
- 核心回归测试：`.\venv\Scripts\python.exe tests/test_core.py`
- 出题/切片/批改测试：`tests/test_api_call.py`、`tests/test_chunking.py`、`tests/test_scoring.py`

## 常见报错判断

- `学科已存在`：说明 SQLite 已有同名学科，换名称或后续增加编辑功能。
- `没有找到可用候选资料`：说明当前搜索源返回为空，建议配置本地 SearXNG。
- `DeepSeek 未返回可用知识点结构`：说明模型返回 JSON 不符合固定结构，应检查 API Key、模型名和提示词。
- Chroma telemetry 报错：当前是依赖兼容噪声，若检索接口仍返回 200，可暂不影响学习功能。

## 遗留问题

- 还没有做人工资料审查队列，只做了登录、付费、验证码 URL 的基础过滤。
- 题目来源入库目前以公开摘要和生成知识点为主，还未实现开放题库来源审查。
- Windows 客户端暂不做，建议等网页链路稳定后封装 PWA 或 PyWebView。
