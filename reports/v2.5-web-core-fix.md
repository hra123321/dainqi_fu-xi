# v2.5 网页核心链路修复报告

## 1. 版本目标

本轮只处理网页核心链路，目标是把“题目生成 → 作答 → 批改 → 错题记录”修到可稳定回归的状态，并把页面中的乱码、题目截断和答案外泄风险一起收掉。

## 2. 本轮完成项

### 2.1 出题结构化

- 将出题流程改为严格 JSON 结构输出
- 删除旧版按空行拆题的脆弱逻辑
- 增加 JSON 解析失败自动重试一次
- 复杂题、多小问题干作为单题整体保存

### 2.2 服务端题目会话

- 新增 `app/services/question_session_service.py`
- 题目标准答案、参考答案、解析只保存在服务端
- 前端提交时只上传 `question_id` 和学生答案

### 2.3 页面修复

- 重写 `app/templates/base.html`
- 重写 `app/templates/index.html`
- 重写 `app/templates/exam.html`
- 重写 `app/templates/wrong_book.html`
- 重写 `app/static/js/app.js`
- 修复中文乱码和学习路径指引不清问题

### 2.4 错题本与 Skill 安全

- 错题记录补充学科、知识点、题型
- Skill 自进化改为候选文件验证机制
- 没有专项测试时禁止覆盖源码
- 候选文件必须通过语法检查、专项测试、核心回归

### 2.5 测试补齐

- 重写 `tests/test_core.py`
- 新增 `tests/test_scoring.py`
- 新增 `tests/test_chunking.py`
- 新增 `tests/test_api_call.py`

## 4. 发现的问题与结论

### 4.1 已解决

- 复杂题会被拆成多题
- 第三小问可能丢失
- 前端能提前拿到参考答案
- 页面学科、按钮、标题出现乱码
- 错题本直接拼 HTML，存在内容注入风险

### 4.2 未在本轮解决

- `knowledge-first` skill 仍指向其他项目知识库
- Chroma telemetry 兼容告警仍存在
- 浏览器自动化实测被本机安全策略阻断

## 5. 下一轮建议

1. 先修本项目独立知识库路径和 Chroma 依赖版本
2. 再补页面端真实浏览器截图级验证
3. 然后继续做教材导入、知识点召回质量和错题聚类分析
