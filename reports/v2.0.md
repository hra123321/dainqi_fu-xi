## 调用到的 Skills

### 1. knowledge-first (本地向量知识库优先检索)
- **调用方式**: 项目开始时自动搜索本地向量知识库获取上下文
- **作用**: 在刷题时从 Chroma 召回关联教材原文作为参考上下文
- **对应实现**: app/vector_db/retriever.py → retriever.search()

### 2. create-plan (计划生成)
- **调用方式**: 制定分阶段实施方案
- **作用**: 将需求拆分为 3 个阶段 12+ 个具体步骤

### 3. aspnet-core (ASP.NET Core 技能)
- **调用方式**: 参考 Web 后端架构模式设计 FastAPI 路由
- **作用**: 路由组织结构参考了 MVC 模式 (router/services/models)

### 4. playwright (浏览器自动化)
- **调用方式**: 参考浏览器控制思路
- **作用**: 前端页面调试时通过浏览器验证渲染结果

### 5. pdf (PDF 处理)
- **调用方式**: 使用 PyMuPDF + pdfplumber 提取课件文字
- **作用**: PDF 上传→文字提取→切片→向量化入库

### 6. datasheet_reader (数据手册读取)
- **调用方式**: 参考 PDF 解析模式
- **作用**: 为未来扩展电气元件数据手册检索做准备

### 7. mcp-builder (MCP 服务构建)
- **调用方式**: 参考 API 设计模式设计 RESTful 接口
- **作用**: 统一 API 路由设计规范

### 8. gh-address-comments / gh-fix-ci (GitHub 协作)
- **调用方式**: Git 版本管理 + 远程推送
- **作用**: 每次改动规范提交 GitHub，重大迭代生成报告

### 9. browser:control-in-app-browser (浏览器调试)
- **调用方式**: 启动 Edge 浏览器预览前端页面
- **作用**: 前端启动自动调用 Edge，手动测试页面渲染

### 10. planner (实施规划)
- **调用方式**: 将项目拆分为可执行的任务列表
- **作用**: 制定 3 阶段实施计划和每个步骤的检查点
